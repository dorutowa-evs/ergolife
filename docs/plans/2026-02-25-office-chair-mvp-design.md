# 办公椅筛选与对比 PoC（MVP）设计文档

日期：2026-02-25
状态：Draft（待 Review）

## 1. 目标与摘要

本 PoC 目标是在桌面端验证办公椅选购中的两个核心价值：

1. 高效筛选：通过条件过滤快速缩小候选商品。
2. 参数对比：将多个商品加入对比并进行横向比较。

MVP 仅覆盖办公椅单品类，使用前端内置样本数据（20+ 型号），不依赖后端服务，不做登录系统。

## 2. 范围定义

### 2.1 In Scope

1. 三个页面：
   1. 筛选/浏览页（`/chairs`）
   2. 详情页（`/chairs/[id]`）
   3. 对比页（`/compare`）
2. 三个“加入对比”入口：
   1. 筛选页商品卡片按钮
   2. 详情页按钮
   3. 对比页“添加商品”按钮（弹窗搜索并选择）
3. 对比页操作：
   1. 添加商品
   2. 删除商品
   3. 拖拽排序
4. 筛选字段：
   1. 价格
   2. 材质
   3. 颜色
   4. 头枕
   5. 腰靠
   6. 腰靠可调节
5. 持久化：筛选条件与对比列表在刷新后保留。

### 2.2 Out of Scope

1. 移动端适配
2. 多品类家具
3. 用户登录与跨设备同步
4. 真实电商 API 接入
5. 生产级监控与权限体系

## 3. 用户与成功标准

### 3.1 目标用户

普通消费者（个人选购办公椅）。

### 3.2 成功标准（MVP DoD）

1. 三页完整链路可稳定演示。
2. 三个“加入对比”入口都可用。
3. 对比页可增删、可拖拽排序。
4. 页面刷新后状态恢复正确。
5. 桌面端交互清晰、无关键阻塞缺陷。

## 4. 信息架构与流程

### 4.1 页面结构

1. `/chairs`
   1. 左侧：筛选面板（sticky）
   2. 右侧顶部：结果数 + 价格排序下拉
   3. 右侧内容：商品卡片网格（3 列）
   4. 右下角：悬浮对比按钮（FAB）
2. `/chairs/[id]`
   1. 商品基础信息（图、名、价）
   2. 核心参数
   3. 图文卖点
   4. 加入对比按钮
3. `/compare`
   1. 顶部操作区（添加商品）
   2. 对比列表（删除、拖拽排序）
   3. 参数对比表
   4. 空状态引导

### 4.2 筛选页（`/chairs`）设计细节

#### 整体布局

Header/Navigation 固定在顶部，下方区域左右分割：

- 左侧筛选面板：固定宽度约 260px，页面滚动时保持 sticky 可见
- 右侧顶部工具栏：左侧显示"共 X 件商品"，右侧价格排序下拉（低→高 / 高→低）
- 右侧内容区：3 列卡片网格，筛选条件变化时实时更新
- 无结果时：右侧显示空状态（图标 + 文案 + "重置筛选"按钮）
- 右下角：悬浮对比按钮（FAB）

#### 筛选面板

从上到下字段顺序，面板顶部标题行右侧有"重置全部"按钮：

| 字段 | 控件 | 清除方式 |
|------|------|----------|
| 价格 | 双端滑条（range slider），滑条下方显示当前区间数值 | × 重置至全区间 |
| 材质 | 多选 Checkbox 列表 | × 取消所有勾选 |
| 颜色 | 色块 + 颜色名称，多选 | × 取消所有勾选 |
| 头枕 | 三态按钮组：`不限 \| 有 \| 无`，默认"不限" | × 回到"不限" |
| 腰靠 | 三态按钮组：`不限 \| 有 \| 无`，默认"不限" | × 回到"不限" |
| 腰靠可调节 | 三态按钮组：`不限 \| 有 \| 无`，默认"不限" | × 回到"不限" |

所有条件变化实时触发筛选，无需点"应用"按钮。

#### 商品卡片

- 默认态：图片（4:3 比例，居中裁切）+ 商品名称 + 价格
- hover 态（图片区）：图片上浮现半透明深色遮罩，遮罩中央显示操作按钮：
  - 未加入对比：显示"＋ 加入对比"（白色按钮）
  - 已加入对比：显示"✓ 移出对比"（区别样式）
- 点击卡片非按钮区域：跳转详情页

#### 悬浮对比按钮（FAB）

- 位置：固定在页面右下角
- 默认态：胶囊/圆形按钮，显示对比图标 + "对比 (N)"，对比列表为空时隐藏
- hover 态：向上展开小列表，每行显示商品名称 + × 直接移除
- 点击按钮主体：在新标签页打开 `/compare`

### 4.3 主流程

1. 用户在 `/chairs` 筛选并浏览结果。
2. 用户从任一入口加入对比。
3. 用户进入 `/compare` 继续添加、删除、排序并查看参数。
4. 刷新页面后保持当前筛选和对比状态。

## 5. 技术方案

1. 框架：Next.js + TypeScript
2. 样式：Tailwind CSS
3. 数据源：前端静态 JSON（20+ 办公椅）
4. 状态管理：React Context + Hooks（PoC 级别）
5. 本地持久化：`localStorage`（后续可切换 IndexedDB）

## 6. 交互细节与边界

1. 重复添加同一商品：不重复入列，给“已在对比中”反馈。
2. 对比列表为空：展示空状态并引导去筛选页/详情页添加。
3. 本地存储读取失败：自动回退默认状态，避免白屏。
4. 字段缺失：表格显示 `--` 占位。
5. 拖拽排序失败：保持原顺序并提示重试。

## 7. 测试场景（PoC 必测）

1. 单条件与多条件筛选结果正确。
2. 三个入口加入对比都可用。
3. 对比页删除功能正常。
4. 对比页拖拽排序后顺序正确持久化。
5. 刷新后筛选条件和对比列表恢复正确。
6. 列表页、详情页、对比页数据一致。
7. 异常存储数据下应用可恢复。

## 8. Review 清单（请你重点确认）

1. 对比页字段展示顺序：是否保持与筛选顺序一致。
2. 对比页”添加商品”弹窗：是否需要带当前已筛选条件。
3. 详情页图文卖点：每个商品建议 2-4 条是否合适。

## 9. 当前默认决策（如无异议将按此实现）

1. 只做桌面端。
2. 办公椅 20+ 样本前端内置。
3. 不登录，不接后端 API。
4. 对比列表有数量上限（具体值见各页面产品文档）。
5. 对比支持拖拽排序。
6. 筛选与对比状态均持久化到 `localStorage`。
7. 筛选页所有条件变化实时触发，无"应用"按钮。
8. 价格筛选使用双端滑条（range slider）。
9. 材质多选 checkbox，颜色色块 + 文字多选。
10. 头枕/腰靠/腰靠可调节使用三态按钮组（不限 / 有 / 无）。
11. 卡片仅展示图片 + 名称 + 价格，"加入/移出对比"按钮在 hover 图片时显示。
12. 筛选页价格排序仅支持低→高 / 高→低，后续可扩展。
13. 对比入口为右下角 FAB，hover 展开对比列表预览，点击在新标签页打开对比页。
