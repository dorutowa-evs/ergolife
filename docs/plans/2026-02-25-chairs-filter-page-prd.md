---

## 1. 页面目标

让用户通过多维条件快速缩小候选椅子范围，并从此页直接发起对比。

---

## 2. 用户目标

- 我想按价格、材质、颜色、头枕/腰靠等条件筛选，快速找到符合预期的椅子。
- 我想把感兴趣的椅子加入对比列表，随时管理、随时跳转对比页查看。
- 我刷新页面后，不希望丢失已设置的筛选条件和对比列表。

---

## 3. 页面布局

```
┌─────────────────────────────────────────────────────┐
│  Header（固定顶部）                                   │
├────────────────┬────────────────────────────────────┤
│                │  共 X 件商品          [价格排序 ▾]  │
│  筛选面板       ├────────────────────────────────────┤
│  (sticky,      │  □□□  □□□  □□□  □□□                 │
│   ~260px)      │  □□□  □□□  □□□  □□□  ← auto 列网格  │
│                │  （列数随容器宽度自适应）              │
└────────────────┴────────────────────────────────────┘
                                        ┌──────────┐
                                        │ ⚖ 对比(N)│ ← FAB（右下角固定）
                                        └──────────┘
```

- Header 固定在顶部（sticky），显示 `ErgoLife` 与导航占位项
- 左侧筛选面板：宽约 260px，页面滚动时 sticky 保持可见
- 右侧内容区：顶部工具栏 + 卡片网格
- 右下角：悬浮对比按钮（FAB），对比列表为空时隐藏

#### Header 占位规范（MVP）

- 左上角品牌文案使用占位：`ErgoLife`
- 导航区展示两个占位项：`Office Chairs`、`Accessories`
- 右上角展示占位项：`Login` 与 `CartIcon`
- 导航与右上角占位项当前仅用于展示，不配置实际跳转路径
- 本页不定义 active 高亮规则，后续与全站导航统一时补充

## 4. 功能规格

### 4.1 筛选面板

#### 面板顶部

- 标题"筛选条件"
- 右侧"重置全部"按钮：有任意筛选条件激活时显示，点击重置所有字段至默认值

#### 筛选字段

| 字段 | 控件类型 | 默认值 | 单独清除方式 |
|------|----------|--------|-------------|
| 价格 | 双端滑条 + 两个可编辑输入框（滑条与输入框双向联动） | 默认全区间（min/max 从 `chairs.catalog.json` 的 `config.priceMin` / `config.priceMax` 读取） | × 重置至全区间 |
| 材质 | 多选 Checkbox 列表 | 全不选（不限） | × 取消所有勾选 |
| 颜色 | 色块（圆形）多选，无文字标签 | 全不选（不限） | × 取消所有勾选 |
| 头枕 | 三态按钮组：`不限 \| 有 \| 无` | 不限 | × 回到"不限" |
| 腰靠 | 三态按钮组：`不限 \| 有 \| 无` | 不限 | × 回到"不限" |
| 腰靠可调节 | 三态按钮组：`不限 \| 有 \| 无` | 不限 | × 回到"不限" |

**颜色字段说明：** 每个颜色选项仅显示圆形色块，不显示文字标签。颜色列表（含颜色名称与 RGB 值）从 `chairs.catalog.json` 读取，支持多选。

**三态按钮说明：**
- `不限`：不过滤该字段，显示所有商品
- `有`：仅显示具备该功能的商品
- `无`：仅显示不具备该功能的商品

#### 价格输入框交互规则

- 滑条拖动 → 实时更新两个输入框数值
- 输入框输入 → 失焦（blur）或按 Enter 后更新滑条位置并触发筛选
- `min > max` 时：两个输入框红色高亮，输入框下方显示错误提示文字，不触发筛选
- 输入框清空时：视为该端恢复配置边界值（min 恢复 `config.priceMin`，max 恢复 `config.priceMax`）
- 输入值超出配置范围时：自动 clamp 至合法边界

#### 筛选触发方式

所有字段变化实时触发筛选，右侧卡片网格即时更新，无"应用"按钮。

#### 筛选逻辑规则

- 同一字段内多选使用 `OR` 逻辑（示例：材质选择网布 + 皮革时，命中任一材质即可）
- 不同字段之间使用 `AND` 逻辑（示例：需同时满足价格、材质、颜色等已选条件）
- 材质与颜色筛选项从 `chairs.catalog.json` 动态生成
- 材质与颜色筛选项展示顺序遵循 JSON 内定义顺序

### 4.2 商品卡片网格

#### 工具栏（网格顶部）

- 左侧：`共 X 件商品`（X 随筛选结果实时更新）
- 右侧：排序下拉菜单，选项：
  - 默认排序
  - 价格从低到高
  - 价格从高到低

**排序规则补充：**
- 默认排序：按原始数据顺序
- 价格排序遇到同价商品时：保持原始数据顺序
- 排序状态会持久化，刷新后恢复上次选择

#### 卡片设计

**默认态：**
- 图片（4:3 比例，居中裁切）
- 商品名称
- 价格

**Hover 态（鼠标悬停图片区域时）：**
- 图片上出现半透明深色遮罩
- 遮罩中央显示操作按钮：
  - 该商品未加入对比 → 显示"＋ 加入对比"
  - 该商品已加入对比 → 显示"✓ 移出对比"（样式与前者有视觉区别）

**点击行为：**
- 点击卡片（非按钮区域）：跳转至详情页 `/chairs/[id]`
- 点击"加入对比"：将该商品加入对比列表
- 点击"移出对比"：将该商品从对比列表移除

#### 空状态

筛选结果为 0 件时，右侧内容区显示：
- 空状态图标
- 文案：`没有符合条件的商品`
- 副文案：`试试调整筛选条件`

#### 网格布局

- CSS Grid `auto-fill + minmax`，列数由容器宽度自动决定
- 最小列宽从 `chairs.catalog.json` 配置（如 `minColumnWidth: 280px`）
- 每页显示数量从 `chairs.catalog.json` 配置（如 `itemsPerPage: 12`），与列数无关

#### 分页

- 位置：网格底部居中
- 组件：`← 上一页` + 页码列表 + `下一页 →`
- 页码展示规则（仿 UI 图）：
  - 总页数 ≤ 5：显示全部页码
  - 总页数 > 5：显示首页、末页、当前页及其相邻页，其余用 `...` 省略，示例：`← 1 2 3 ... 8 →`
- 当前页高亮显示
- 第 1 页时"上一页"禁用，最后一页时"下一页"禁用
- 切页仅更新当前商品列表，不重置筛选与排序状态
- 筛选或排序变化后，如当前页超出总页数，自动回到第 1 页
- 筛选结果为 0 件时不显示分页

### 4.3 悬浮对比按钮（FAB）

**位置：** 页面右下角固定（fixed）

**默认态（有商品在对比列表时显示）：**
- 对比图标 + "对比" 文字 + 数量徽标（显示当前对比列表商品数）
- 对比列表为空时：FAB 隐藏

**Hover 态：**
- 向上展开对比列表预览
- 每行显示商品名称 + `×` 移除按钮
- 点击 `×` 直接从对比列表移除对应商品，列表实时更新

**点击 FAB 主体：** 在新标签页打开 `/compare`

#### 对比数量与预览规则

- 对比列表全局上限为 `5` 个商品（筛选页、详情页、对比页入口统一生效）
- 当尝试添加第 `6` 个商品时：阻止添加，显示 Toast `最多对比 5 个商品`；卡片 hover 按钮仍正常显示"＋ 加入对比"，点击时触发 Toast
- FAB hover 预览列表按加入时间顺序展示（旧 -> 新）
- 鼠标离开 FAB 主体与预览区域后，预览立即收起

## 5. 交互规则与边界

| 场景 | 行为 |
|------|------|
| 重复添加同一商品 | 不重复入列；卡片 hover 态已显示"✓ 移出对比"作为反馈 |
| 对比列表已满 5 个时继续添加 | 阻止添加，不改动已有列表，显示 Toast `最多对比 5 个商品`；卡片按钮保持正常样式 |
| 筛选结果为空 | 显示空状态，不报错 |
| 本地存储读取失败 | 自动回退默认筛选状态，不白屏 |
| 点击"重置全部" | 仅重置筛选字段，不影响对比列表 |
| 点击"重置全部"后 | 立即将重置后的筛选状态写回 `localStorage` |
| 切换分页 | 仅更新当前页商品，不重置筛选与排序 |
| 筛选/排序变化导致分页越界 | 当前页自动回到第 1 页 |
| 刷新页面 | 筛选条件、排序状态、分页状态与对比列表从 `localStorage` 恢复 |

---

## 6. 持久化

- 筛选条件持久化到 `localStorage`（key: `chair-filter`）
- 对比列表持久化到 `localStorage`（key: `compare-list`）
- 排序状态持久化到 `localStorage`（key: `chair-sort`）
- 分页当前页持久化到 `localStorage`（key: `chair-page`）
- 读取失败时静默回退至默认值
